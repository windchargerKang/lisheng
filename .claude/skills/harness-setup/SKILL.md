# Harness Suite 初始化

## 目的

自动探索项目背景，生成个性化配置。最小化用户输入。

---

## Step 1: 自动探索项目

### 静默检测

```
========================================
  正在探索你的项目...
========================================

  ├─ 检测项目类型...
  ├─ 分析目录结构...
  ├─ 识别技术栈...
  ├─ 扫描现有文档...
  └─ 生成个性化配置...

  [按任意键跳过探索]
```

### 检测内容

| 检测项 | 方式 | 输出 |
|--------|------|------|
| 项目类型 | 检测 pom.xml/package.json/build.gradle | Spring/FE/Android/iOS/其他 |
| 目录结构 | ls -la src/ | 模块列表 |
| 技术栈 | 扫描依赖文件 | 框架、ORM、测试框架 |
| 现有文档 | 检测 docs/ | 已有知识库 |
| 构建命令 | 检测构建文件 | mvn/npm/gradle |

## Step 2: 用户确认

### 两种选择

```
┌─────────────────────────────────────────┐
│                                         │
│  检测结果是否符合你的预期？               │
│                                         │
│    [确认]  → 使用以上配置，生成文档      │
│    [讨论]  → 告诉我需要修改的地方       │
│                                         │
└─────────────────────────────────────────┘

  ↑↓ 选择    Enter 确认
```

### "确认"路径

```
→ 跳过所有问题，直接生成文档
→ 输出简洁的成功信息
```

### "讨论"路径

```
┌─────────────────────────────────────────┐
│  请告诉我需要修改的部分：               │
│                                         │
│  [1] 项目类型（当前：Spring Boot）       │
│  [2] 模块结构                           │
│  [3] 技术栈                             │
│  [4] 高风险区域                         │
│  [5] 其他（自由输入）                   │
│                                         │
│  输入编号或直接描述：                   │
└─────────────────────────────────────────┘
```

### 讨论后的选择

```
修改完成后的选择：

  [确认生成]  使用修改后的配置生成文档
  [继续讨论]  还有其他需要修改的
```

## 命令行选项

```
/harness-setup              交互式初始化（推荐）
/harness-setup --auto       完全自动，使用检测结果
/harness-setup --force      覆盖已有文件
/harness-setup --docs-only  仅生成文档，跳过 openspec
/harness-setup --help       显示帮助
```

## 关键文件说明

### CLAUDE.md
```
位置：项目根目录
用途：项目技术规约
重要性：⭐⭐⭐⭐⭐

包含：
- 架构分层规则
- 测试要求
- 事务规范
- 受保护路径
```

### AGENTS.md
```
位置：项目根目录
用途：代理行为规范
重要性：⭐⭐⭐⭐⭐

包含：
- 工作流程
- 必读文件清单
- 命令速查
```

### REVIEW.md
```
位置：项目根目录
用途：评审标准
重要性：⭐⭐⭐⭐

包含：
- 评审检查项
- OpenSpec 对齐验证
```

### docs/ 知识库
```
用途：项目知识沉淀

维护方式：
- 手动维护重要规范
- /harness-apply 和 /harness-review 自动捕获知识
- /harness-knowledge 管理知识条目
```
