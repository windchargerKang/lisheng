# 需求提案：用户地址管理功能

## 背景

在移动端购物流程中，用户需要在结算页面填写收货地址。当前实现存在以下问题：

1. **地址无法复用**：每次下单都需要重新输入地址，用户体验差
2. **地址无管理入口**：用户无法提前维护多个收货地址
3. **无默认地址支持**：无法设置常用地址为默认，提高下单效率

为了提升购物体验，需要建立独立的地址管理模块，支持用户添加、编辑、删除和选择收货地址。

## 目标

1. **地址管理页面**
   - 地址列表展示
   - 新增/编辑/删除地址
   - 设置默认地址

2. **结算页联动**
   - 从地址列表选择收货地址
   - 自动填充默认地址
   - 支持在结算流程中新增地址

3. **数据持久化**
   - 地址与用户账号绑定
   - 支持多地址管理

## 非目标

- 不涉及智能地址解析（如 OCR 识别名片）
- 不涉及区域选择器（省市区三级联动）
- 不涉及地址共享/分享给他人

## 利益相关方

| 角色 | 说明 |
|------|------|
| C 端用户 | 维护个人收货地址，下单时选择 |
| 店铺 | 接收订单时获取用户收货地址 |
| 运营人员 | 查看订单收货信息管理 |

## 验收标准

1. 用户能够新增收货地址（姓名、电话、地址为必填）
2. 用户能够编辑已有地址
3. 用户能够删除地址（软删除）
4. 用户能够设置一个默认地址
5. 结算页面显示当前选择的地址
6. 点击地址栏可跳转到地址列表选择
7. 选择地址后自动填充到结算表单
8. 默认地址在列表顶部显示并有标记

## 备注

- 地址删除采用软删除，保留历史数据
- 默认地址只有一个，设置新默认时自动取消其他
- 结算页通过 `selectMode=1` 参数进入地址选择模式

## 扩展需求（实现过程中追加）

- 订单详情页支持修改收货地址（仅限待确认订单）
- 订单详情页支持修改订单备注（仅限待确认订单）
