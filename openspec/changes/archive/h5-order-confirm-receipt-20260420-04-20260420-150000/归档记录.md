## 需求摘要

### 背景
运营管理端发货功能已完成，订单状态可以流转到"已发货"（shipped）。但 H5 端用户收到货物后，缺少确认收货的功能，导致订单无法完成最终的状态流转。

### 目标
1. 订单详情页确认收货入口 - 已发货状态的订单显示"确认收货"按钮
2. 订单状态流转 - shipped (已发货) → completed (已完成)
3. 后端 API 支持 - 新增确认收货 API 端点

## 技术方案摘要

### 接口变更
| 方法 | 路径 | 权限 | 说明 |
|------|------|------|------|
| POST | /api/v1/orders/{id}/confirm | 用户权限 | 确认收货 |

### 前端变更
- h5/src/views/OrderDetail.vue - 添加确认收货按钮和二次确认弹窗

## 实现摘要

### 后端实现
1. 新增确认收货 API 端点（orders.py）
   - 验证订单属于当前用户
   - 验证订单状态为 shipped
   - 更新状态为 completed
2. 编写 4 个测试用例（test_orders.py）
   - 确认收货成功测试
   - 无效状态测试
   - 他人订单测试
   - 不存在订单测试

### 前端实现
1. 修改订单详情页（OrderDetail.vue）
   - 添加确认收货按钮（仅 shipped 状态显示）
   - 二次确认弹窗
   - 确认后刷新订单状态
