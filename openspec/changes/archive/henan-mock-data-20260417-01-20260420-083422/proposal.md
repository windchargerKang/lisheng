# 需求提案：河南省模拟数据生成

## 背景

当前系统的模拟数据初始化脚本 (`init_mock_data.py`) 中的区域数据较为简单，仅包含 24 个基础区域。为了更好地测试和演示渠道销售管理系统的多级渠道管理、区域分润计算等功能，需要生成一套以河南省为核心的完整模拟数据。

## 目标

1. 生成河南省完整的省市区三级区域数据（18 个地级市 + 县级区）
2. 基于河南省区域生成区代、店铺、客户的多级渠道关系
3. 生成与河南省区域绑定的采购订单和客户订单
4. 生成符合河南省场景的分润记录

## 非目标

- 修改现有价格体系和分润规则
- 新增业务功能或 API
- 修改数据库表结构

## 利益相关方

| 角色 | 说明 |
|------|------|
| 开发团队 | 用于功能测试和演示 |
| 测试团队 | 用于集成测试和回归测试 |
| 运营人员 | 用于培训和学习系统使用 |

## 验收标准

1. 河南省 18 个地级市完整录入（郑州、开封、洛阳、平顶山、安阳、鹤壁、新乡、焦作、濮阳、许昌、漯河、三门峡、南阳、商丘、信阳、周口、驻马店、济源）
2. 每个地级市至少包含 3-5 个县级区
3. 生成至少 18 个区代（每个地级市 1 个）
4. 生成至少 50 个店铺（分布在不同区域）
5. 生成至少 100 个客户（分布在不同区域）
6. 生成至少 30 笔客户订单（覆盖不同区域和状态）
7. 生成至少 10 笔采购订单（供应商发货到区代/店铺）
8. 生成至少 20 条分润记录（客户/店铺/区代/主播）
9. 所有数据关联正确的区域 ID

## 备注

- 数据来源：参考河南省行政区划代码
- 数据生成脚本放置于 `backend/scripts/init_henan_mock_data.py`
- 可独立运行，不影响现有 `init_mock_data.py` 脚本
