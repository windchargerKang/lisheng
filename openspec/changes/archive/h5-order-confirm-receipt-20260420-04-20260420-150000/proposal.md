# 需求提案：H5 端订单确认收货功能

## 背景

运营管理端发货功能已完成，订单状态可以流转到"已发货"（shipped）。但 H5 端用户收到货物后，缺少确认收货的功能，导致订单无法完成最终的状态流转（shipped → completed）。

## 目标

1. **订单详情页确认收货入口**
   - 已发货状态的订单显示"确认收货"按钮
   - 点击按钮后确认收到货物

2. **订单状态流转**
   - shipped (已发货) → completed (已完成)

3. **后端 API 支持**
   - 新增确认收货 API 端点
   - 验证订单属于当前用户
   - 验证订单状态为已发货

## 非目标

- 不涉及退货/退款流程
- 不涉及订单评价功能
- 不涉及自动确认收货（超时自动确认）

## 利益相关方

| 角色 | 说明 |
|------|------|
| C 端用户 | 收到货物后确认收货，完成订单 |
| 运营管理员 | 查看已完成订单统计 |

## 验收标准

1. 用户能够查看已发货订单的详情
2. 已发货状态的订单显示"确认收货"按钮
3. 点击确认收货后，订单状态更新为"已完成"
4. 只有已发货状态的订单可以确认收货
5. 用户只能确认自己的订单
6. 确认后 H5 端订单列表和详情页状态正确显示

## 备注

- 确认收货是不可逆操作，需要二次确认
- 后续可扩展自动确认收货功能（如发货后 15 天自动确认）
