# 需求提案：供应商管理

## 背景

随着业务规模扩大，公司需要管理多个供应商的采购、入库、结算等业务。目前系统仅面向 C 端用户和渠道代理，缺少对上游供应商的管理能力，导致：

1. 采购订单通过 Excel 或微信沟通，无法追踪
2. 入库与库存数据不同步
3. 供应商结算周期长，对账困难
4. 无法分析供应商供货质量和效率

## 目标

1. 建立供应商档案管理系统
2. 实现采购订单全流程管理（创建→入库→结算）
3. 与库存系统打通，自动更新库存
4. 提供供应商对账和结算功能
5. 支持供应商端门户（Web 端）

## 非目标

- 供应商移动端 APP（第一阶段仅 Web 端）
- 智能采购建议（如自动补货算法）
- 供应商评级系统（可在后续迭代添加）
- 财务系统对接（独立结算流程）

## 利益相关方


| 角色   | 说明               |
| ---- | ---------------- |
| 采购员  | 创建采购订单、跟踪入库、发起结算 |
| 库管员  | 采购入库、质检、库存管理     |
| 财务人员 | 供应商对账、结算审核、付款    |
| 供应商  | 查看订单、确认供货、对账收款   |
| 管理员  | 供应商资质审核、权限管理     |


## 验收标准

1. 能够创建和管理供应商档案（资质、联系方式、结算方式）
2. 能够创建采购订单并跟踪状态
3. 采购入库后自动更新库存数量
4. 能够生成供应商对账单
5. 供应商能够通过独立账号登录查看订单和对账
6. 采购流程与现有产品/库存数据打通

## 备注

- 供应商端可考虑使用独立域名或子域名部署
- 需要与现有渠道销售系统数据隔离
- 第一阶段支持现款结算和账期结算两种模式
