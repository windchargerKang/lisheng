# 需求提案：店铺地图定位配置与查看

## 背景

当前系统中店铺只有文字地址信息，用户无法直观地了解店铺的地理位置。为了提升用户体验，需要为店铺增加地图坐标属性，让用户可以在地图上查看附近的店铺位置。

## 目标

1. 为店铺增加地图坐标（经纬度）属性
2. 管理员可以在运营管理端配置/修改每个店铺的地图坐标
3. H5 端"我的"页面增加"查看附近的店铺"菜单
4. 用户可以在地图上查看自己附近的所有店铺位置

## 非目标

- 不提供店铺导航路线规划（后续可扩展）
- 不提供店铺预约、排队等功能
- 不涉及店铺覆盖范围/配送范围的绘制

## 利益相关方

- **系统管理员**：需要在运营端配置店铺坐标
- **所有 H5 端用户**：可以查看附近店铺
- **店铺运营人员**：坐标配置后会影响用户找店的体验

## 验收标准

1. 数据库中店铺表增加经纬度字段（latitude, longitude）
2. 运营端店铺管理页面可以编辑店铺的地图坐标（支持手动输入或在地图上选点）
3. H5 端"我的"页面显示"查看附近的店铺"菜单入口
4. 点击菜单后打开地图页面，显示用户当前位置和附近的店铺标记
5. 点击店铺标记可以查看店铺基本信息（名称、地址、联系方式等）

## 备注

- 地图服务选择：建议使用高德地图或百度地图（需确认现有项目使用的地图 SDK）
- 隐私考虑：获取用户位置前需要明确提示并获取授权
- 性能考虑：店铺列表查询需要按距离排序，可能需要数据库支持地理位置查询
