# 需求提案：手机号登录功能

## 背景

当前系统仅支持账号密码登录方式。为了提升用户体验，降低注册门槛，需要增加手机号登录方式。用户可以自由选择账号登录或手机号登录，两种登录方式并行存在。

## 目标

1. 在 H5 登录页面增加手机号登录方式，与账号登录二选一
2. 手机号登录表单仅需填写：手机号、密码、滑块验证
3. 登录页面底部增加注册按钮入口
4. 后端支持手机号登录验证
5. 参考提供的 UI 设计图实现页面布局

## 非目标

1. 本次不包含短信验证码登录（仅密码登录）
2. 本次不包含手机号注册功能（注册按钮跳转到独立注册页面）
3. 本次不包含一键获取本机号码登录

## 利益相关方

- **H5 用户**：可以使用手机号快捷登录
- **运营人员**：需要管理手机号账号体系
- **开发团队**：需要实现前后端登录逻辑

## 验收标准

1. H5 登录页面支持"账号"和"手机"两种登录方式切换
2. 手机号登录表单包含：手机号输入框（带 +86 选择）、密码输入框、滑块验证
3. 手机号登录需要勾选"我已阅读并同意登录协议与隐私权政策"
4. 登录按钮下方有"忘记密码"和"注册"两个入口
5. 后端 API 支持手机号登录验证（手机号 + 密码）
6. 滑块验证功能正常工作
7. 登录成功后跳转首页，失败时显示明确错误提示

## 备注

- UI 参考用户提供的截图设计
- 滑块验证可使用 Vant 4 的 `van-captcha` 组件或第三方滑块验证服务
- 手机号格式校验需符合中国大陆手机号规则（11 位数字）
