# 需求提案：移动端购物全流程打通

## 背景

当前 H5 移动端已具备基础的用户、商品、购物车、订单等模块，但各功能之间的联动尚未完全打通：

- 首页缺少推荐产品列表展示
- 产品详情页下单功能未与购物车联动
- 购物车结算后生成订单的流程需要完善
- 支付流程需要集成

为了实现完整的移动端购物体验，需要将首页 → 产品列表 → 产品详情 → 购物车 → 下单 → 支付 → 订单的全流程串联起来。

## 目标

1. **首页推荐产品列表**
  - 展示推荐商品（新品、热销）
  - 支持点击跳转到产品详情
2. **产品下单联动购物车**
  - 产品详情页支持"加入购物车"和"立即购买"
  - 购物车支持商品数量修改、删除、选中结算
3. **支付后生成订单**
  - 结算页面确认订单信息
  - 集成支付方式（模拟支付）
  - 支付成功后生成正式订单
4. **订单管理**
  - 订单列表查看
  - 订单详情查看
  - 订单状态流转（待支付 → 已支付 → 已完成）

## 非目标

- 不涉及后台管理端的订单管理功能
- 不涉及真实支付渠道集成（使用模拟支付）
- 不涉及物流跟踪功能
- 不涉及退款/售后流程

## 利益相关方


| 角色        | 说明                   |
| --------- | -------------------- |
| 客户（C 端用户） | 浏览商品、加入购物车、下单支付、查看订单 |
| 店铺        | 接收订单、发货              |
| 区代        | 查看区域内订单、获得分润         |
| 运营人员      | 管理商品上下架、订单处理         |


## 验收标准

1. 首页能够正确展示推荐产品列表（至少显示 4 个商品）
2. 产品详情页能够正确展示商品信息，支持加入购物车
3. 购物车能够正确展示已添加的商品，支持数量修改和删除
4. 结算页面能够正确计算订单总金额
5. 模拟支付成功后生成订单，订单状态正确流转
6. 订单列表能够查看用户的所有订单
7. 订单详情能够查看订单完整信息和商品明细

## 备注

- 支付环节使用模拟支付，不涉及真实支付渠道
- 优先保证核心购物流程的完整性
- 后续迭代再考虑优惠券、满减等营销功能
