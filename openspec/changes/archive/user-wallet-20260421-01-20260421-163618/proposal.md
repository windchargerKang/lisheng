# 需求提案：用户钱包系统

## 背景



当前系统中用户账号只有基本的身份信息，没有独立的资金账户概念。为了支持后续的业务发展（如代理商充值、店铺充值、提现等），需要为每个用户建立一个钱包账户，实现资金的独立管理。

## 目标



1. 为每个用户自动创建钱包账户
2. 支持钱包充值功能（记录充值流水）
3. 支持钱包提现功能（记录提现流水、审核流程）
4. 钱包余额查询
5. 充值/提现记录查询

## 非目标



1. 本次不包含支付渠道对接（如微信支付、支付宝、银行卡）
2. 本次不包含钱包转账功能
3. 本次不包含资金冻结/解冻功能
4. 本次不包含复杂的财务对账功能

## 利益相关方



1. **系统管理员** - 需要审核提现申请、查看充值/提现记录
2. **区代/店铺用户** - 使用钱包充值、提现功能
3. **普通用户** - 可能使用钱包功能（取决于业务规则）
4. **财务/运营人员** - 查看资金流水、对账

## 验收标准

1. 新用户注册时自动创建钱包账户，初始余额为 0
2. 充值接口能够正确增加钱包余额并记录流水
3. 提现接口能够正确扣减钱包余额并记录流水，状态为待审核
4. 提现审核通过后，状态变更为已完成
5. 提现审核拒绝后，状态变更为已拒绝，余额退回
6. 用户只能查看自己的钱包信息和流水记录
7. 管理员可以查看所有用户的钱包信息和审核提现

## 备注



- 需要设计钱包表（wallets）和流水表（wallet_transactions）
- 需要考虑事务一致性问题（余额更新与流水记录）
- 需要考虑并发安全问题（同一钱包同时充值/提现）
