# 需求提案：H5 用户端

## 背景

第一阶段已完成 Web 运营端基础框架，现在需要开发 H5 用户端，让终端用户能够通过移动端浏览器访问系统，进行产品浏览、下单购买、查看收益等操作。

## 目标

1. 实现 H5 移动端用户界面
2. 用户登录/注册功能
3. 产品浏览和详情展示
4. 购物车和订单功能
5. 个人中心和收益查看
6. 分享推广功能

## 非目标

- 原生 App 开发（iOS/Android）
- 小程序开发
- 支付功能集成（第一阶段仅做订单，不涉及实际支付）

## 利益相关方

| 角色 | 说明 |
|------|------|
| 客户 | 通过 H5 端浏览产品、下单购买、查看收益 |
| 店铺 | 查看店铺数据、下级客户消费情况 |
| 区代 | 查看区域数据、下级店铺/客户情况 |
| 公司运营方 | 监控 H5 端运营数据 |

## 技术选型

| 技术 | 选择 | 说明 |
|------|------|------|
| 框架 | Vue 3 | 与 Web 运营端保持一致 |
| UI 库 | Vant 4 | 移动端组件库 |
| 状态管理 | Pinia | 与 Web 端一致 |
| 构建工具 | Vite 5 | 快速构建 |

## 功能模块

### 1. 用户模块
- 登录/注册
- 忘记密码
- 个人信息管理
- 角色切换（客户/店铺/区代）

### 2. 产品模块
- 产品列表（分页、筛选）
- 产品详情
- 价格展示（根据用户角色显示对应价格）

### 3. 购物车模块
- 添加/删除商品
- 修改数量
- 结算

### 4. 订单模块
- 订单列表（全部/待付款/已完成）
- 订单详情
- 订单状态跟踪

### 5. 收益模块
- 收益总览
- 收益明细
- 提现申请
- 提现记录

### 6. 分享模块
- 生成分享链接/二维码
- 分享记录
- 下级列表

## 验收标准

1. H5 端能够正常登录（复用后端认证 API）
2. 产品列表和详情能够正确展示
3. 用户角色不同，显示的价格不同（零售价/店铺价/区代价）
4. 能够创建订单并查看订单状态
5. 能够查看个人收益明细
6. 能够生成分享链接

## 备注

- H5 端需要适配主流移动端浏览器
- 需要考虑移动端网络不稳定场景
- 分享功能需要与微信等平台兼容
