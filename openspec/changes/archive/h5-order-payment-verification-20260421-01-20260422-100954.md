## 需求摘要

实现 H5 端订单支付与核销流程，包含两套业务模式：

1. **核销模式（消费者）**：下单 → 钱包支付 → 生成核销码 → 店铺核销 → 分润结算
2. **电商模式（店铺/区代）**：下单 → 钱包支付 → 订单流转 → 不涉及核销

## 技术方案摘要

### 数据模型变更
- Order 表新增 `order_type`、`verified_at` 字段
- OrderStatus 新增 `VERIFIED` 状态
- 创建 VerificationCode 核销码表
- WalletTransaction 新增交易类型：ORDER_PAYMENT、SERVICE_FEE、AGENT_PROFIT

### 服务层实现
- OrderService：订单创建、确认（钱包扣款）
- VerificationCodeService：核销码生成和验证
- ProfitService：分润计算和发放（30% 服务费 +10% 区代利润）

### API 变更
- POST /orders/create：根据用户角色自动设置订单类型
- POST /orders/{id}/confirm：钱包扣款确认订单
- POST /verification/verify：店铺核销接口

### H5 前端
- 创建 Verification.vue 核销页面
- 修改 Payment.vue 使用 /confirm 接口触发钱包扣款
- 移除收益总览相关路由（与钱包功能重复）
