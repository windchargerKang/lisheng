# 知识管理

## 用途

管理和维护项目中捕获的隐性知识、坑点、规范等。

## 使用方式

```
/harness-knowledge              查看所有知识
/harness-knowledge add          添加新知识
/harness-knowledge edit <id>    编辑指定知识
/harness-knowledge clean        清理过时知识
/harness-knowledge export       导出知识到文件
```

## /harness-knowledge add

### 添加新知识

```
[添加知识]

请选择知识类型：
  [1] 隐性约定（前后端/模块间口头约定）
  [2] 架构坑点（已发现的架构问题）
  [3] 接口规范（接口返回/参数约定）
  [4] SQL 规范（数据库使用规范）
  [5] 产品规则（业务规则）
  [6] 其他

请输入编号：_
```

### 填写知识详情

```
[添加知识 - 隐性约定]

内容描述：
  _______________________________________

来源位置：
  _______________________________________
  （文件:行号 或 模块名）

为什么重要：
  _______________________________________
  （描述潜在风险或依赖方）

标签（逗号分隔）：
  _______________________________________

┌─────────────────────────────────────┐
│  [保存] 保存到 implicit-contracts.md │
│  [预览] 预览效果                    │
│  [取消] 放弃                        │
└─────────────────────────────────────┘
```

     添加时间：2026-03-15
     最近引用：2026-04-05（4天前）

┌─────────────────────────────────────┐
│  [自动清理] 清理从未引用的条目       │
│  [逐个审查] 逐个确认是否清理         │
│  [取消] 退出                        │
└─────────────────────────────────────┘
```

  [3] 仅 architecture
  [4] 自定义选择

  [默认: docs/knowledge-export.md]

┌─────────────────────────────────────┐
│  [导出] 开始导出                    │
│  [取消] 退出                        │
└─────────────────────────────────────┘
```

## 知识索引结构

| 类型 | 存储文件 | 标签示例 |
|------|---------|---------|
| 隐性约定 | `implicit-contracts.md` | 前后端约定、状态语义、默认值 |
| 架构坑点 | `architecture/index.md` | 循环依赖、调用链路、模块边界 |
| 接口规范 | `specs/api.md` | 返回结构、参数校验、幂等性 |
| SQL 规范 | `standards/database.md` | 索引规范、NOLOCK、分页 |
| 产品规则 | `product/index.md` | 业务规则、错误码、流程约束 |

## 与其他 Skill 的关系

- `harness:apply` - 执行时捕获知识
- `harness:review` - 评审时捕获知识
- `harness:knowledge` - 管理和维护知识
