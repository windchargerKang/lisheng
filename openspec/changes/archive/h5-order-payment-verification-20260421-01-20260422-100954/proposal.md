# 需求提案：H5 端订单支付与核销流程

## 背景

在已完成的钱包系统基础上，需要完善 H5 端下单后整个订单和资金的流程。当前系统已具备：

- 用户钱包、充值、提现功能
- 订单管理基础功能
- 购物车功能

但订单支付、资金流转、核销流程尚未打通，需要实现两套不同的业务流程。

## 目标

### 流程一：消费者下单 + 店铺核销（核销模式）

1. **下单阶段**：消费者选择商品 → 生成订单到购物车 → 订单状态为"待确认"
2. **确认支付**：用户点击确认 → 从用户钱包扣除金额 → 金额转入 lisheng 账号（系统管理员）钱包 → 订单状态变为"已完成"
3. **生成核销码**：已完成的订单自动生成核销码
4. **店铺核销**：H5 端店铺账号使用核销功能核销订单
5. **分润结算**：
  - 服务费：订单金额的 30% 从 lisheng 账号钱包扣除，转入核销店铺钱包
  - 区代利润：订单金额的 10% 从 lisheng 账号钱包扣除，转入核销店铺的绑定区代钱包

### 流程二：店铺/区代下单（普通电商模式）

1. **下单阶段**：店铺/区代选择商品 + 填写收货地址 → 生成订单到购物车 → 订单状态为"待确认"
2. **支付**：从用户钱包扣除金额 → 金额转入 lisheng 账号钱包 → 订单状态流转
3. **订单流转**：待确认 → 待发货 → 已发货 → 已完成
4. **利润优惠**：利润直接优惠到价格上，不涉及核销流程

## 非目标

- 不修改现有购物车基础功能
- 不修改现有订单列表展示功能
- 不涉及退款流程（后续迭代）
- 不涉及部分核销、批量核销

## 利益相关方


| 角色             | 影响                |
| -------------- | ----------------- |
| 消费者（C 端用户）     | 使用核销模式下单、支付、获取核销码 |
| 店铺账号           | 可核销消费者订单、获得服务费分润  |
| 区代账号           | 获得绑定店铺核销订单的利润分润   |
| 系统管理员（lisheng） | 接收订单金额、支出分润       |
| 运营管理端          | 查看订单流水、分润记录       |


## 验收标准

1. **消费者下单流程**
  - 消费者可选择商品并确认下单
  - 订单初始状态为"待确认"
  - 用户确认后从钱包扣除对应金额
  - 扣除金额转入 lisheng 账号钱包
  - 订单状态变为"已完成"并生成核销码
2. **店铺核销流程**
  - 店铺账号可在 H5 端输入核销码核销订单
  - 核销时自动计算服务费（订单金额 30%）
  - 服务费从 lisheng 钱包转入店铺钱包
  - 核销时自动计算区代利润（订单金额 10%）
  - 区代利润从 lisheng 钱包转入绑定区代钱包
  - 核销后订单状态更新为"已核销"
3. **店铺/区代普通下单流程**
  - 店铺/区代下单时需填写收货地址
  - 支付流程与消费者一致（钱包扣款 → lisheng 收款）
  - 订单状态正常流转（待确认 → 待发货 → 已发货 → 已完成）
  - 不涉及核销码生成
4. **资金流水**
  - 所有钱包变动均有流水记录
  - 流水类型区分：订单支付、服务费返还、区代利润
  - 管理员可查看全部分润流水
5. **并发安全**
  - 同一订单不能重复核销
  - 钱包扣款需保证原子性和余额充足检查
