## 需求摘要

为系统所有用户创建钱包账户，支持管理员手动充值和用户提现申请功能。

**核心功能：**
1. 新用户注册时自动创建钱包账户
2. 支持钱包充值功能（记录充值流水）
3. 支持钱包提现功能（记录提现流水、审核流程）
4. 钱包余额查询
5. 充值/提现记录查询

**利益相关方：**
- 系统管理员 - 审核提现申请、查看充值/提现记录
- 区代/店铺用户 - 使用钱包充值、提现功能

## 技术方案摘要

**新增模块：**
- `backend/app/models/wallet.py` - 钱包数据模型
- `backend/app/models/wallet_transaction.py` - 钱包流水数据模型
- `backend/app/api/v1/wallet.py` - 钱包 API 路由
- `backend/app/schemas/wallet.py` - 钱包请求/响应 Schema
- `backend/app/services/wallet_service.py` - 钱包业务逻辑层

**数据模型：**
- Wallet 表：user_id (唯一索引), balance DECIMAL(10,2)
- WalletTransaction 表：wallet_id, transaction_type (RECHARGE/WITHDRAW), amount, status (PENDING/COMPLETED/REJECTED), transaction_no (唯一)

**流水号规则：** `{R/W}{YYYYMMDD}{4 位序号}`

**关键技术决策：**
- 使用 FOR UPDATE 行锁防止并发提现
- 可用余额 = 钱包余额 - 待审核提现总额
- 提现采用一级审核流程

## 实现摘要

**后端实现：**
- WalletService 服务类实现核心业务逻辑
- 充值 API：管理员可给任何用户充值，普通用户只能给自己充值
- 提现 API：用户申请提现，状态为 PENDING
- 审核 API：管理员审核提现申请（通过/拒绝）
- 流水查询 API：支持按类型、状态筛选

**前端实现：**
- H5 端 (`h5/src/views/Wallet.vue`)：用户钱包首页，支持充值、提现、查询流水
- 管理端 (`frontend/src/views/WalletAdmin.vue`)：钱包管理，支持代客充值、提现审批
- 路由配置：H5 端 `/wallet`，管理端 `/wallet` (我的钱包) 和 `/wallet-admin` (钱包管理)

**测试覆盖：**
- 8 个单元测试覆盖充值、提现、审核、并发控制等场景

**修复问题：**
- 流水号生成日期格式不匹配 bug（`%Y%m%d` vs `%Y-%m-%d`）

## 评审结论

**严重问题（已修复）：**
- P0: 并发提现可能导致余额超扣 → 已添加 FOR UPDATE 行锁和待审核金额计算

**警告问题（已修复）：**
- 用户 - 钱包关系缺失 → 已添加 ForeignKey 和双向关系
- 流水号生成逻辑日期格式不匹配 → 已修复查询条件

**索引建议：**
- 现有索引满足需求，暂无新增索引需求
