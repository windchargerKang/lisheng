# 需求提案：供应商门户

## 背景

供应商管理系统第一阶段已完成运营端内嵌的供应商档案、采购订单、入库和结算管理。但目前供应商无法自主查看订单、对账和收款信息，仍需通过人工沟通，导致：

1. 供应商无法实时查看采购订单状态
2. 对账需人工发送对账单，效率低
3. 供应商无法自主确认供货信息
4. 付款进度不透明，供应商频繁咨询

## 目标

1. 建立供应商独立门户（Web 端）
2. 供应商可查看采购订单并确认供货
3. 供应商可自主对账和查看结算进度
4. 供应商可维护企业资质和联系方式
5. 与现有采购系统数据打通

## 非目标

- 供应商移动端 APP（仅 Web 端）
- 供应商在线投标/报价功能
- 供应商评级和考核系统
- 与财务系统对接

## 利益相关方

| 角色 | 说明 |
|------|------|
| 供应商 | 查看订单、确认供货、对账收款 |
| 采购员 | 发布订单、跟踪供应商确认 |
| 财务人员 | 结算审核、付款 |
| 管理员 | 供应商账号和权限管理 |

## 验收标准

1. 供应商能够通过独立账号登录门户
2. 供应商可查看采购订单并确认/拒绝供货
3. 供应商可查看入库记录和结算进度
4. 供应商可维护企业资质和联系方式
5. 门户与运营端数据实时同步

## 备注

- 考虑使用独立域名或子域名部署（如 supplier.xxx.com）
- 供应商账号与运营端账号体系隔离
- 第一阶段仅支持已合作的供应商（由管理员创建账号）
