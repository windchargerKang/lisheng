# SQL 风险审查

## 输入

`$ARGUMENTS` = 本次改动涉及的 SQL、Mapper、XML、Repository 文件

## 检查项

### 1. 无条件更新/删除检查
- 是否存在没有 WHERE 条件的 UPDATE
- 是否存在没有 WHERE 条件的 DELETE
- 批量操作是否有限制条件

### 2. 批量操作范围检查
- 批量更新是否有清晰的范围控制
- 批量删除是否有人为可控的上限
- 是否可能影响大量数据

### 3. 分页稳定性检查
- 分页查询是否有稳定的排序条件
- 没有排序的分页可能导致数据重复或遗漏

### 4. 扫描范围检查
- 查询条件是否可能导致全表扫描
- 索引是否被正确使用
- LIKE 前缀匹配是否影响索引

### 5. N+1 查询检查
- 循环中是否存在数据库查询
- 一对多/多对多查询是否可能产生 N+1

### 6. 索引影响说明
- 新增/修改的 SQL 是否影响索引
- 是否需要创建新索引
- 索引变更是否需要数据迁移

## 输出格式

```markdown
## SQL 风险评审

### 严重问题（必须修复）
1. [文件:行号] 问题描述
2. [Mapper XML] 风险 SQL

### 警告问题（建议修复）
1. [文件:行号] 问题描述

### 建议项
1. 优化建议

### 索引影响
- 新增索引：
- 禁用索引：
```

## 规则

- 只读操作，不修改任何文件
- 重点关注数据安全性和性能风险
- 对于不熟悉的表结构，标注「需确认」
