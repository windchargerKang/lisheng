# 需求提案：用户权限管理

## 背景

随着系统用户数量增加和角色类型多样化（普通用户、店铺代理、区代、运营管理员、供应商等），当前简单的 role_type 字段已无法满足复杂的权限管理需求：

1. 缺少细粒度的权限控制（菜单级、按钮级、数据级）
2. 无法灵活配置角色权限
3. 缺少操作日志审计
4. 管理员账号管理分散，无统一管理界面
5. 无法实现数据隔离（如区代只能查看本区数据）

## 目标

1. 建立基于 RBAC（角色 - 权限）的权限管理体系
2. 实现菜单级、按钮级权限控制
3. 实现数据级权限（区域、部门等维度）
4. 提供角色和权限的统一管理界面
5. 记录用户操作日志，支持审计
6. 支持账号的创建、禁用、密码重置等功能

## 非目标

- 单点登录（SSO）集成
- 第三方认证（OAuth、LDAP）
- 多因素认证（MFA）
- 权限申请审批流程

## 利益相关方

| 角色 | 说明 |
|------|------|
| 系统管理员 | 配置角色权限、管理账号 |
| 运营人员 | 使用分配的业务权限 |
| 区代/店铺代理 | 查看权限范围内的数据 |
| 供应商 | 仅能访问供应商门户 |
| 审计人员 | 查看操作日志 |

## 验收标准

1. 支持自定义角色并配置权限
2. 权限控制到菜单和按钮级别
3. 支持数据范围权限（如区代只能查看本区数据）
4. 提供账号管理界面（创建、编辑、禁用、重置密码）
5. 记录关键操作日志（登录、增删改等）
6. 支持按角色筛选和查看用户列表

## 备注

- 优先实现运营端权限管理，H5 端保持简单角色判断
- 供应商门户权限独立管理，不与运营端混用
- 操作日志需保留至少 6 个月
