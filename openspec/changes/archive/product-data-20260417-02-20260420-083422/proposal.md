# 需求提案：河南省特色商品数据生成

## 背景

当前系统的产品数据仅有 10 个通用商品，缺乏与河南省区域特色相关的商品数据。为了更好地展示渠道销售系统在地方特色商品场景下的应用能力，需要生成一批河南省特色商品数据。

## 目标

1. 生成 30-50 个河南省特色商品（食品、农产品、工艺品等）
2. 商品分类覆盖：特产食品、生鲜农产品、传统工艺品、保健品等
3. 每个商品包含完整的三级价格体系（零售价/店铺价/区代价）
4. 商品关联到河南省的供应商

## 非目标

- 修改产品价格体系规则
- 新增商品分类或属性字段
- 修改数据库表结构

## 利益相关方

| 角色 | 说明 |
|------|------|
| 运营人员 | 管理商品上下架、价格调整 |
| 店铺 | 采购并销售商品 |
| 区代 | 管理区域内商品推广 |
| 供应商 | 提供商品货源 |

## 验收标准

1. 生成至少 30 个河南省特色商品
2. 每个商品包含完整的三级价格
3. 商品 SKU 代码唯一
4. 商品关联到河南省供应商
5. 商品分类合理（特产/生鲜/工艺品/保健品）
6. 脚本可重复运行（数据已存在时跳过）

## 备注

- 商品数据应体现河南特色（如：信阳毛尖、新郑红枣、焦作山药等）
- 价格应符合市场合理区间
- 可考虑在现有 `init_henan_mock_data.py` 脚本中扩展，或创建独立脚本
