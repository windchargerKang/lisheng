# 需求提案：产品分润比例配置

## 背景

当前系统的分润比例（服务费 30%、区代利润 10%）是全局统一配置，存储在 `backend/config.json` 文件中。这种设计存在以下问题：

1. **灵活性不足**：无法针对不同产品设置不同的分润比例
2. **运营需求**：实际业务中，不同品类的产品利润率不同，需要差异化分润
3. **配置不便**：修改分润比例需要编辑配置文件并重启服务

## 目标

1. 支持在产品管理页面为每个产品单独配置分润比例
2. 分润计算时优先使用产品级配置，未配置时使用全局默认值
3. 提供友好的 UI 界面供运营人员配置

## 非目标

1. 不修改现有的全局默认分润配置（作为兜底）
2. 不涉及历史订单分润的追溯调整

## 利益相关方

- **运营管理人员**：需要在产品管理页面配置分润比例
- **财务/老板**：关注分润收益的准确性

## 验收标准

1. 产品管理页面新增"服务费比例"和"区代利润比例"配置项
2. 分润计算时按产品配置的比例执行
3. 产品未配置分润比例时，使用全局默认值（服务费 30%、区代利润 10%）
4. 分润比例支持小数，精确到 0.01%（如 15.50%）
5. 分润比例有合理范围限制（0% - 100%）

## 备注

- 分润比例字段需要支持 nullable，为空时使用全局配置
- 前端需要添加输入校验，防止错误配置
