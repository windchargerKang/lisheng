# 技术方案：手机号登录功能

## 方案概述

在现有账号登录基础上增加手机号登录方式，采用统一账号体系（手机号作为用户的另一种登录标识），前端实现 H5 登录页面改造和独立注册页面，后端支持手机号登录和注册验证。

## 详细设计

### 架构变更

**前端模块：**
- `h5/src/views/Login.vue` - 登录页面改造，增加账号/手机 Tab 切换
- `h5/src/views/Register.vue` - 新建独立注册页面
- `h5/src/router/index.ts` - 新增注册页面路由

**后端模块：**
- `backend/app/api/v1/auth.py` - 登录/注册 API 改造
- `backend/app/schemas/auth.py` - 登录/注册请求 Schema 调整
- `backend/app/models/user.py` - User 模型新增 phone_number 字段

### 数据模型变更

**User 表新增字段：**
```python
phone_number = Column(String(20), unique=True, index=True, nullable=True)
```

- 允许为空（兼容现有账号）
- 唯一索引（一个手机号只能绑定一个账号）
- 长度 20（包含国际区号，如 +86）

### 接口变更

**登录接口（现有接口改造）：**
```
POST /auth/login
Request: { "username": "手机号或用户名", "password": "密码" }
```
- `username` 字段同时支持用户名或手机号登录
- 后端自动识别：11 位数字按手机号匹配，否则按用户名匹配

**注册接口（新增）：**
```
POST /auth/register-by-phone
Request: {
  "phone_number": "13800138000",
  "password": "密码",
  "confirm_password": "确认密码"
}
Response: { "token": "...", "user": {...} }
```

## 影响范围

- **受影响模块**：认证模块、用户模型、H5 登录/注册页面
- **受保护路径变更**：无
- **向后兼容性**：兼容现有账号登录，不影响已注册用户

## 风险评估

| 风险 | 等级 | 缓解方案 |
|------|------|----------|
| 手机号重复注册 | 中 | 数据库唯一索引 + 注册前校验 |
| 滑块验证被绕过 | 中 | 后续可升级为第三方验证码服务 |
| 手机号格式校验 | 低 | 前端 + 后端双重校验 |
| 现有用户无手机号 | 低 | phone_number 允许为空，不影响 |

## 事务与数据

- **事务边界**：注册操作需在事务中完成（创建用户 + 初始化钱包）
- **数据迁移**：无需迁移（现有用户 phone_number 为空即可）
- **回滚方案**：注册失败时回滚所有数据库操作
