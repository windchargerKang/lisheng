# 需求提案：运营管理端订单发货功能

## 背景

当前订单管理系统中，H5 端用户已完成下单和确认订单功能，订单状态流转到"待发货"（confirmed）。但运营管理端缺少发货功能，导致：

1. **无法发货操作**：管理员无法填写物流信息并将订单状态更新为"已发货"
2. **订单流程中断**：订单停留在"待发货"状态，无法流转到"已发货"和"已完成"
3. **用户无法确认收货**：缺少发货环节，用户端也无法实现确认收货功能

为了实现完整的订单购物流程，需要在运营管理端实现发货功能。

## 目标

1. **订单列表页**
  - 查看所有订单列表
  - 按订单状态筛选（待发货、已发货等）
  - 查看订单详情
2. **发货操作**
  - 填写物流公司信息
  - 填写物流单号
  - 更新订单状态为"已发货"
3. **订单状态流转**
  - confirmed (待发货) → shipped (已发货)

## 非目标

- 不涉及多个包裹分开发货（一个订单只支持一次发货）
- 不涉及物流轨迹跟踪
- 不涉及批量发货
- 不涉及退款/售后处理

## 利益相关方


| 角色    | 说明            |
| ----- | ------------- |
| 运营管理员 | 在管理后台操作发货     |
| 仓库人员  | 根据订单拣货打包      |
| C 端用户 | 查看订单发货状态，确认收货 |


## 验收标准

1. 管理员能够查看所有订单列表
2. 管理员能够筛选"待发货"状态的订单
3. 管理员能够查看订单详情（商品信息、收货地址）
4. 管理员能够填写物流公司和物流单号
5. 发货后订单状态更新为"已发货"
6. 发货后 H5 端用户能够看到物流信息
7. 只有"待发货"状态的订单可以执行发货操作

## 备注

- 物流公司可以先用文本输入，后续可改为下拉选择
- 物流单号为必填项
- 发货操作需要记录操作人和操作时间
