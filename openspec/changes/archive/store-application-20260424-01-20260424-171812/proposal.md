# 需求提案：H5 端"我想开店"申请功能

## 背景
当前 H5 端用户如果想成为店铺或区代角色，缺乏便捷的申请渠道。需要在 H5 端提供一个"我想开店"入口，允许普通客户直接在线提交升级申请，简化用户升级流程，降低运营沟通成本。

## 目标
1. 在 H5 端"我的"页面增加"我想开店"入口
2. 提供两种升级选择：店铺申请、区代申请
3. 店铺申请需填写：店铺名称、区域、对应区代（自动关联出区域对应的区代，不可编辑）、地图定位坐标
4. 区代申请需填写：区代名称、区域（需重复性检验）、推荐区代（选填）
5. 提交后进入待审核状态，用户可查看详情
6. 管理员审核通过后自动升级账号角色
7. 审核拒绝后账号保持普通客户身份，允许重新申请
8. 运营管理端提供审核功能：查看申请列表、审核通过/拒绝、填写拒绝理由

## 非目标
1. 不包含店铺/区代的后续管理功能
2. 不包含申请费用支付流程（如有）

## 利益相关方
- **普通客户**：可以便捷提交升级申请
- **管理员**：在运营管理端审核申请
- **现有店铺/区代**：可能作为被关联的"对应区代"或"推荐区代"

## 验收标准
1. [ ] "我的"页面可见"我想开店"入口
2. [ ] 用户可选择申请类型（店铺/区代）
3. [ ] 店铺申请表单包含：店铺名称、区域选择、对应区代选择、地图定位选点
4. [ ] 区代申请表单包含：区代名称、区域选择（需校验重复性）、推荐区代选择（选填）
5. [ ] 提交成功后显示"待审核"状态
6. [ ] 用户可查看申请进度和审核结果
7. [ ] 审核通过后账号角色自动升级
8. [ ] 审核拒绝后可重新提交申请
9. [ ] 同一用户同一时间只能有一个待审核申请
10. [ ] 运营管理端可查看申请列表
11. [ ] 运营管理端可审核通过/拒绝，拒绝时需填写理由

## 备注
- 地图定位可复用现有 NearbyShops 页面中的高德地图集成
- 区域重复性检验需要后端 API 支持
- 需要后端提供申请提交、状态查询、角色升级、审核等接口
